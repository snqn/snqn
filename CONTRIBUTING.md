# Contributing

Thanks a ton for taking the time to contribute!

Open Source thrives on collaboration!

- Made a port? Show me! Submit a PR! I *want* to see it.
- Unsure where to start? Feel free to contact me. (I don't bite.. usually.)
- Found an issue? Tell me! No gatekeeping here.

\-\-\-

## The Contribution Guide:

- Make sure to stick to the core values written in the [README](/README.md#philosophy).
- Consider using [conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
- When creating READMEs, follow the same schema that my other ports use.

### Naming Convention

Use either `sine qua non` or `snqn` where it makes sense.

There's really not much more to it.
